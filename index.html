多线程交替打印字符串问题记录

问题背景

leetcode.1195. 交替打印字符串

编写一个可以从 1 到 n 输出代表这个数字的字符串的程序，但是：

  • 如果这个数字可以被 3 整除，输出 “fizz”。
  • 如果这个数字可以被 5 整除，输出 “buzz”。
  • 如果这个数字可以同时被 3 和 5 整除，输出 “fizzbuzz”。

例如，当 n = 15，输出： 1, 2, fizz, 4, buzz, fizz, 7, 8, fizz, buzz, 11, fizz, 13, 14, fizzbuzz。

假设有这么一个类：

请你实现一个有四个线程的多线程版 FizzBuzz， 同一个 FizzBuzz 实例会被如下四个线程使用：

  1. 线程A将调用 fizz() 来判断是否能被 3 整除，如果可以，则输出 fizz。
  2. 线程B将调用 buzz() 来判断是否能被 5 整除，如果可以，则输出 buzz。
  3. 线程C将调用 fizzbuzz() 来判断是否同时能被 3 和 5 整除，如果可以，则输出 fizzbuzz。
  4. 线程D将调用 number() 来实现输出既不能被 3 整除也不能被 5 整除的数字。

尚为科技面试

提问

1.spring框架下文件上传大小限制的原因和方法

spring会限制上传的文件大小I，这个限制其实限制在spring内嵌的tomcat上，即限制了tomcat的最大吞吐量max-swallow-size,我们可以在spring的配制文件中设置为自己想要的大小spring:servlet:max-file-size,max-request-size,当上传的文件大小超出配置的大小时，spring会抛出MaxUploadSizeExceedException异常。

2.登录功能的实现原理

cookie-session,jwt,其中cookie-session可以将数据存放在redis当中，避免分布式多节点造成的数据冗余和同步压力，而jwt不存放后端数据，但是去需要花费cpu时间进行解密，相当于时间换空间

3.redis的数据结构

string,hash,set,zset,list,bitmap,geo,hypeloglog,stream

4.redis怎么设置一个永不过期的键，他的默认过期时间是多少

傻逼面试官，我说了redis默认就是永不过期的，还跟我扯这么多，查资料发现就是默认永不过期

5.redis过期键回收策略

三种，惰性删除，定期删除，内存淘汰机制，惰性删除就是当读到一个过期的键时就会直接删除掉这个键，定期删除就是redis会主动淘汰一批过期的key,内存淘汰机制就是当使用过的内存超出最大可用内存限定时，就会主动清理

6.redis持久化的方式

三种方式，RDB（快照），AOF（只追加文件，命令日志），RDB和AOF混合

我的优惠券页面

  1. 顶部导航栏
    • 显示页面标题“我的优惠券”。
    • 包含返回按钮。
  2. 选项卡导航
    • 包含四个选项卡：全部、即将过期、平台券、多多支付优惠。
    • 用户可以点击切换不同的优惠券分类。
    • 选项卡应适应不同屏幕尺寸，保持易用性。
  3. 优惠券列表
    • 显示用户当前拥有的优惠券列表。
    • 每个优惠券项包含优惠券金额、使用条件、有效期和操作按钮。
    • 列表应支持下拉刷新和加载更多功能。
  4. 推荐好券
    • 显示推荐给用户的优惠券列表。
    • 每个推荐项包含商品图片、商品名称、优惠券金额、券后价格、领取进度和操作按钮。
    • 推荐列表应适应不同屏幕尺寸，保持布局美观。
  5. 按钮组件
    • 用于执行操作，如“立即使用”、“立即领取”等。
    • 按钮大小和间距应适应不同屏幕尺寸，确保易点击。
  6. 图片组件
    • 用于显示商品图片。
    • 图片应自适应容器大小，保持比例。
  7. 文本组件
    • 用于显示优惠券金额、使用条件、有效期、商品名称、券后价格、领取进度等信息。
    • 文本大小和行距应适应不同屏幕尺寸，确保可读性。
  8. 链接组件
    • 用于引导用户获取更多优惠券。
    • 链接样式应适应不同屏幕尺寸，保持一致性。

交互功能

  1. 选项卡切换
    • 用户可以点击不同的选项卡查看不同分类的优惠券。
  2. 优惠券使用
    • 用户可以点击“立即使用”按钮使用优惠券。
  3. 优惠券领取
    • 用户可以点击“立即领取”按钮领取推荐优惠券。
  4. 查看更多优惠券
    • 用户可以点击“领更多好券”链接查看更多优惠券。

非功能需求

  1. 响应式设计
    • 页面需要适配不同尺寸的屏幕，包括手机、平板和PC。
    • 使用百分比、flexbox或grid布局实现动态调整。
  2. 性能优化
    • 页面加载速度需要优化，确保用户体验流畅。
    • 图片和资源应进行压缩和懒加载。
  3. 兼容性
    • 页面需要兼容主流的浏览器和操作系统。
  4. 安全性
    • 确保用户数据的安全性和隐私保护。

领券中心页面

页面布局：

  1. 搜索栏
    • 位于页面顶部，用户可以输入关键词搜索优惠券。
  2. 分类导航
    • 包含多个分类标签，如精选、家电数码、家居百货、食品生鲜、美家等，方便用户快速找到所需商品类别。

java的类加载器

三种类加载器，启动类加载器，扩展类加载器，应用程序类加载器

双亲委派机制有什么用

1.防止类被重复加载，每个类加载器加载一个类时都会询问父类有没有加载过这个类，如果父类没有加载这个类的话，那么就由自己加载这个类，这样就不会出现如果每个类加载器都自己加载类就会出现重复加载的情况

2.防止对一些不能修改类进行修改，比如Object类，如果不遵守双亲委派机制，那么我们就可以使用自己的类加载器加载我们自己编写的Object类，尽管这个已经被加载了，这样就造成了Object类被修改或者说被替代了

3.判断两个类是否相同，两个类是否相同不仅看他们的全名是否相同，还要看加载他们的累加器是否相同，如果他们的全名相同但是类加载器不同，那么就认为这两个类不相同，毕竟上一条已经说了，被其他类加载器加载有可能会被修改
